Introduction

P-adic fields are provided in Nemo by Flint. This allows construction of -adic fields for any prime .

P-adic fields are constructed using the FlintPadicField function. However, for convenience we define

PadicField = FlintPadicField

so that -adic fields can be constructed using PadicField rather than FlintPadicField. Note that this is the name of the constructor, but not of padic field type.

The types of -adic fields in Nemo are given in the following table, along with the libraries that provide them and the associated types of the parent objects.

Library Field Element type Parent type
Flint padic PadicField

All the -adic field types belong to the Field abstract type and the -adic field element types belong to the FieldElem abstract type.

P-adic field constructors

In order to construct -adic field elements in Nemo, one must first construct the -adic field itself. This is accomplished with one of the following constructors.

# Nemo.FlintPadicField — Method.

FlintPadicField(p::Integer, prec::Int)

Returns the parent object for the -adic field for given prime , where the default absolute precision of elements of the field is given by prec.

source

It is also possible to call the inner constructor directly. It has the following form.

FlintPadicField(p::fmpz, prec::Int)

Returns the parent object for the -adic field for given prime , where the default absolute precision of elements of the field is given by prec.

Here are some examples of creating -adic fields and making use of the resulting parent objects to coerce various elements into those fields.

R = PadicField(7, 30)
S = PadicField(ZZ(65537), 30)

a = R()
b = S(1)
c = S(ZZ(123))
d = R(ZZ(1)//7^2)

Apart from coercing elements into the -adic field as above, we offer the following functions.

The construction can be used to construct -adic values of precision by adding it to integer values representing the -adic value modulo as in the examples.

Here are some examples of constructing -adic field elements.

R = PadicField(7, 30)
S = PadicField(fmpz(65537), 30)

a = one(R)
b = zero(S)
c = 1 + 2*7 + 4*7^2 + O(R, 7^3)
d = 13 + 357*ZZ(65537) + O(S, ZZ(65537)^12)
f = ZZ(1)//7^2 + ZZ(2)//7 + 3 + 4*7 + O(R, 7^2)

Beware that the expression 1 + 2*p + 3*p^2 + O(R, p^n) is actually computed as a normal Julia expression. Therefore if \code{Int} values are used instead of Flint integers or Julia bignums, overflow may result in evaluating the value.

Basic functionality

The following basic functionality is provided by the default -adic field implementation in Nemo, to support construction of generic rings over -adic fields. Any custom -adic field implementation in Nemo should provide these functions along with the usual arithmetic operations.

parent_type(::Type{padic})

Gives the type of the parent object of a Flint -adic field element.

elem_type(R::FlintPadicField)

Given the parent object for a -adic field, return the type of elements of the field.

Base.hash(a::padic, h::UInt)

Return a UInt hexadecimal hash of the -adic field element . This should be xor'd with a fixed random hexadecimal specific to the -adic field type. The hash of the representative of the -adic field element (lifted to and the prime for the field, should be xor'd with the supplied parameter h as part of computing the hash.

deepcopy(a::padic)

Construct a copy of the given -adic field element and return it. This function must recursively construct copies of all of the internal data in the given element. Nemo -adic field elements are mutable and so returning shallow copies is not sufficient.

mul!(c::padic, a::padic, b::padic)

Multiply by and set the existing -adic field element to the result. This function is provided for performance reasons as it saves allocating a new object for the result and eliminates associated garbage collection.

addeq!(c::padic, a::padic)

In-place addition. Adds to and sets to the result. This function is provided for performance reasons as it saves allocating a new object for the result and eliminates associated garbage collection.

Given the parent object R for a -adic field, the following coercion functions are provided to coerce various elements into the -adic field. Developers provide these by overloading the call operator for the -adic field parent objects.

R()

Coerce zero into the -adic field.

R(n::Integer)
R(f::fmpz)
R(f::fmpq)

Coerce an integer or rational value into the -adic field.

R(f::padic)

Take a -adic field element that is already in the -adic field and simply return it. A copy of the original is not made.

In addition to the above, developers of custom -adic field types must ensure that they provide the equivalent of the function base_ring(R::FlintPadicField) which should return Union{}. In addition to this they should ensure that each -adic field element contains a field parent specifying the parent object of the -adic field element, or at least supply the equivalent of the function parent(a::padic) to return the parent object of a -adic field element.

b = 7^2 + 3*7^3 + O(R, 7^5)

u = a + ZZ(1)//7^2
v = (ZZ(12)//11)*b
w = c*(ZZ(1)//7)

Comparison

Nemo provides the comparison operation == for -adic field elements. Julia then automatically provides the corresponding != operation. Here are the functions provided.

Function

==(a::padic, b::padic) isequal(a::padic, b::padic)

Note that == returns true if its arguments are arithmetically equal to the minimum of the two precisions. The isequal function requires them to both be the same precision, as for power series.

In addition, the following ad hoc comparisons are provided, Julia again providing the corresponding != operators.

Function

==(a::padic, b::Integer) ==(a::padic, b::fmpz) ==(a::padic, b::fmpq) ==(a::Integer, b::padic) ==(a::fmpz, b::padic) ==(a::fmpq, b::padic)

Here are some examples of comparisons.

R = PadicField(7, 30)

a = 1 + 2*7 + 4*7^2 + O(R, 7^3)
b = 3*7^3 + O(R, 7^5)
c = O(R, 7^3)
d = R(2)

a == 1 + 2*7 + O(R, 7^2)
b == c
isequal(a, b)
c == R(0)
d == R(2)
ZZ(3) == d
ZZ(3)//7 == c

Square root

# Base.sqrt — Method.

sqrt(a::padic)

Return the -adic square root of . We define this only when the valuation of is even. The precision of the output will be precision valuation. If the square root does not exist, an exception is thrown.

source

Here are some examples of taking the square root.

R = PadicField(7, 30)

log(a::padic)

Return the -adic logarithm of . We define this only when the valuation of is zero (but not for ). The precision of the output will be the same as the precision of the input. If the input is not valid an exception is thrown.

# Nemo.teichmuller — Method.

teichmuller(a::padic)

Return the Teichmuller lift of the -adic value . We require the valuation of to be nonnegative. The precision of the output will be the same as the precision of the input. For convenience, if is congruent to zero modulo we return zero. If the input is not valid an exception is thrown.

source

R = PadicField(7, 30)

a = 1 + 7 + 27^2 + O(R, 7^3) b = 2 + 57 + 37^2 + O(R, 7^3) c = 37 + 2*7^2 + O(R, 7^5)

c = exp(c) d = log(a) c = exp(R(0)) d = log(R(1)) f = teichmuller(b)
